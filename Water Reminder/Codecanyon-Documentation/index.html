A) Setup of Project - top

Here we are sharing step by step guide how to open this android project in Android Studio.

  • Download the latest android studio setup file from this location.
  • Once download is completed install android studio over your system.
  • If you are not sure about android studio configuration, please refer over here.
  • Once Android Studio Setup is completed, continue with below specified steps.
    1. Open Android Studio. After that click on "Open an existing Android Studio project".
    2. After that select the Studio file from "Water Drinking Reminder" downloaded folder and then click OK.
    3. Wait until indexing of the "Water Drinking Reminder" project is completed.
    4. After that click on Projects >> app >> java and choose "Screen_Splash" file, you will be able to view android programming code in Java.
    5. After that click res and then click layout and choose screen_splash.xml file. Here your XML file for designing Android UI will be opened.

B) Project Structure - top

By default, Android Studio displays project files in the Android view.

This Android view contains app directory, some of the modules and Gradle Script group

You can add a new module to the project by clicking File > New > New Module.

Within each Android app module, files are shown in the following groups:

  1. manifests - Contains the AndroidManifest.xml file. AndroidManifest.xml file describes the nature of the application and each of its components. It contains all require permission.
  2. java - Contains the Java source code files, separated by package names, including JUnit test code. Click here to know more about JAVA file structure.
  3. assets - Contains the all application's raw asset files like font file, etc...
  4. res - Contains all non-code resources, such as XML layouts, UI strings, and bitmap images. For more information about all possible resource types, see Providing Resources
  5. Gradle Script - Shows all the project's build-related configuration files.
    • build.gradle(module) - This defines the module-specific build configurations.
    • build.gradle(project) - This file is integral to the project, so you should maintain them in revision control with all other source code. Here, you can add new library(dependencies) for this project.

F) Project Resources - top

Resource directories supported for this project resides over app/res/ directory and it contains application resources, such as drawable files, layout files, and UI string. See Application Resources for more information.

  • anim/ - XML files that define animations.
  • color/ - XML files that define a state list of colors.
  • drawable/ - Bitmap files (.png, .9.png, .jpg, .gif) or Custom drawable XML files.
  • mipmap/ - Drawable files for different launcher icon densities.
  • layout/ - XML files that define a user interface layout.
  • values/ - XML files that contain simple values, such as strings, dimension, styles and colors.
  • font/ - Font files with extensions such as .ttf, .otf.
